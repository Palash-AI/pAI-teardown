## SCREENS TO NAVIGATE

### 1. Home Screen (Returning User View)
**Observe:**
- Personalization: Is content tailored to history/preferences/level?
- Immediate value prop: Streak count, recommended content, daily goal visible?
- Entry friction: Taps to core action (ideal: 1-3)
- Visual hierarchy: What's emphasized (streaks, progress, notifications)?
- Time-based messaging: Contextual ("7-day streak!") vs generic?

**Screenshot:** Full home, streak indicators zoomed, CTA buttons

**Framework check:** Hook Model TRIGGER + Fogg MAP → Does app immediately surface reason to return today?

---

### 2. Profile / Progress Page
**Observe:**
- Progress visibility: Lessons/videos/time tracked transparently?
- Milestone celebration: Badges, certificates on completion?
- Historical data: Browsing/learning history accessible?
- Comparison: vs friends, percentile, global average?
- Trend visualization: Weekly/monthly trends shown?
- Goal setting: Can set daily/weekly goals?

**Screenshot:** Full profile, progress section, graphs, achievement listings, goal UI

**Framework check:** HEART Task Success → Can user see concrete progress toward goals?

---

### 3. Streak / Gamification Surfaces
**Observe:**
- Streak visibility: Displayed prominently (home, profile, during activity)?
- Loss mechanics: Reset on 1-day gap? Warning? Grace period?
- Streak recovery: Available? Cost (free/premium/limited)?
- Milestones: Celebrated at 7, 30, 100, 365 days?
- Visual feedback: Animation, sound, notification on streak increase?
- Frequency messaging: How often streak mentioned in-app?

**Screenshot:** Streak counter (multiple locations), reset/loss UX, recovery UI, milestone screens

**Framework check:** Hook Model INVESTMENT → Does streak create sunk-cost and loss-aversion mechanics?

---

### 4. Badge / Achievement Displays
**Observe:**
- Badge catalog: Count total badges (goal: 20+)
- Badge rarity: Common/rare/legendary tiers?
- Display prominence: Profile, home, popup on earn?
- Achievement notifications: Real-time or delayed?
- Badge categories: Streaks, milestones, challenges, social?
- Social display: Shareable to friends/public?

**Screenshot:** Full badge section, 5-10 individual badges, earn notification, rarity indicators

**Framework check:** Hook Model VARIABLE REWARD → Are badges variable (unpredictable) or fixed milestones?

---

### 5. Notification Center & Content
**Observe:**
- Frequency: Notifications per day/week?
- Personalization: Based on user behavior (time patterns, goals)?
- Trigger types: Streaks, reminders, friends, content, social proof, wins?
- Timing: Sent at optimal times (not 6 AM, not midnight)?
- Message variation: Different copy for same event?
- Action clarity: Clear CTA (open, complete, view)?
- Unsubscribe: Can control frequency, type, time window?

**Screenshot:** Full notification center, 5-10 message variants, settings/preferences

**Framework check:** Fogg MAP TRIGGER → Do notifications prompt action without creating friction?

### 6. Saved / Bookmarked Content
**Observe:**
- Bookmark functionality: Icon, menu, dedicated section?
- Accessibility: Taps to save? During or after content?
- Save confirmation: Visual feedback?
- Saved collection visibility: Buried or prominent?
- Organization: Folders, tags, playlists?
- Sync: Across devices?
- Re-engagement use: Saved items pushed for re-engagement?

**Screenshot:** Content with save button, saved library, organization UI, confirmation

**Framework check:** Hook Model INVESTMENT → Does bookmarking create visible investment?

### 7. History / Recently Viewed
**Observe:**
- History availability: Browsing/learning history section?
- Granularity: Item-level or session-level?
- Sortability: Sort by date, type, rating?
- Re-engagement: Can resume from history?
- Timestamps: Reinforce consistency ("Studied yesterday 2 PM")?
- Clearing: Can user delete history?

**Screenshot:** History section, timestamps, resume interaction

**Framework check:** Hook Model INVESTMENT → Does history visibility reinforce time invested?

### 8. Social Features (Friends, Leaderboards, Sharing)
**Observe:**
- Friend system: Add, follow, compete?
- Leaderboard types: Global, regional, friends, weekly?
- Leaderboard metrics: Points, streaks, lessons, time, XP?
- Social proof signals: "5,000 people learned today"?
- Sharing mechanics: Achievements, certificates, badges, progress?
- Share destinations: Social media, messaging, email, link?
- Cohort features: Study groups, classes, teacher-led?
- Commenting: User interaction on achievements?

**Screenshot:** Friends list, leaderboards (multiple views), achievement share dialog, social proof, cohort section

**Framework check:** Hook Model TRIGGER (social FOMO) + Reforge social proof → Are network effects visible and motivating?

### 9. Re-engagement Surfaces (Lapsed User Experience)
**Observe:**
- What does user see after 3+ days away?
- Re-engagement screen/modal present?
- Messaging tone: Guilt vs positive ("Welcome back!")?
- Win-back offer: Streak recovery, bonus points, free premium?
- Friction removal: One-tap restart?
- Streak recovery mechanics: Cost, limits?
- Notification frequency: Surge post-lapse?

**Screenshot:** First screen after 3+ day gap, re-engagement modal, streak recovery UI, notification frequency

**Framework check:** Reforge Lapsed User Recovery → Does app address loss aversion (streak reset) with recovery incentive?

---

### 10. Daily Challenges / Goals
**Observe:**
- Daily goal visibility: Target present (lessons, time, XP)?
- Goal customization: User-settable?
- Goal type: Completion-based (5 lessons) or time-based (20 min)?
- Goal reminder: Notification on goal?
- Completion celebration: Visual celebration?
- Streak tie-in: Goal completion contributes to streak?
- Weekly/monthly goals: Longer-term goals present?
- Difficulty adjustment: Manual or AI-personalized?

**Screenshot:** Daily goal display, progress indicator, completion celebration, settings

**Framework check:** Fogg MAP PROMPT + Hook Model ACTION → Is goal completion the daily core action that loops?

## SCORING RUBRIC

| Score | Label | Criteria |
|-------|-------|----------|
| 9-10 | Duolingo-level | Prominent streaks (home + milestones at 7/30/100/365), 30+ badges (rarity tiers, shareable), multiple leaderboards (global/friends/weekly), social proof visible, re-engagement with incentive, personalized notifications, AI-difficulty goals |
| 7-8 | Strong habit loop | Visible streaks, 20+ badges, functional leaderboards, sharing present, basic re-engagement (1-2 attempts), personalized reminders, clear goals |
| 5-6 | Early gamification | Streaks exist (buried), 5-10 badges, basic leaderboard or friends, some notifications, minimal re-engagement, generic personalization |
| 3-4 | Passive tracking | History/stats visible but not celebrated, 2-5 badge types, no leaderboard, no social, generic notifications, no re-engagement |
| 1-2 | No habit loop | Pure consumption, no streaks, no badges, no progress celebration, no social, no notifications, no re-engagement |

**Scoring method**: Start at 5. Add: +2 prominent streaks, +1 each (10+ badges, leaderboard, sharing, re-engagement, personalized notifications, goals visible). Deduct: -3 hidden streaks, -2 no badges, -2 no social, -2 no re-engagement, -1 generic notifications. Cap 1-10.

## KEY ISSUE PATTERNS

1. **Broken Hook Cycle** → Detect: Incomplete at TRIGGER (no notifications), ACTION (>5 taps), REWARD (fixed, not variable), INVESTMENT (no visible progress). Fix: Map all 4 phases; add missing element.

2. **Weak Trigger Strategy** → Detect: Only generic daily reminder; no personalization by time/behavior; no internal FOMO. Fix: Implement time optimization, behavior-based messaging, friend activity visibility.

3. **Buried Streaks** → Detect: Streak only on profile; no milestone celebrations; hard reset on 1-day gap. Fix: Move to home, add 7/30/100/365 milestones with badges, add grace period or paid recovery.

4. **No Variable Rewards** → Detect: Same badge earned predictably; no surprise bonuses; algorithm absent. Fix: Randomize badge types, add surprise bonuses, implement spaced-repetition reminders.

5. **Social Isolation** → Detect: No friends system, leaderboards, sharing, or "X people" signals. Fix: Add friend leaderboard (weekly), share-to-social on badges, friend activity notifications.

6. **Missing Re-engagement** → Detect: User 3+ days lapsed sees no modal, no win-back offer, no streak recovery. Fix: Add re-engagement screen on Day 4, offer streak recovery + bonus, show "Welcome back" celebration.

7. **No Progress Visibility** → Detect: Progress tracked only as buried watch-time; no lesson count, no trends, no milestones. Fix: Add progress ring to home, show lessons-per-week, celebrate 25/50/100 lesson milestones.

8. **Passive Consumption Loop** → Detect: User watches/reads, no personalization, no goal, no investment, no streaks. Fix: Add bookmark feature, visible lesson progress bar, daily goal, streak for consistency.
