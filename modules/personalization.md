# Personalization & Relevance

> AARRR: Engagement/Retention | Frameworks: HEART, Hook, Reforge, JTBD, Fogg, Kano | Speed: 20min | Rigor: 60min

## SCREENS TO NAVIGATE

### 1. Onboarding Personalization Probes
**Observe:**
- Questions present: yes / no / optional
- Question type: JTBD-based ("What's your goal?") or demographic (age/location)?
- Number of questions asked
- Impact visibility: does first content screen change based on answers?
- Acknowledgment: "Based on your selection, we found X" or silent change?
- Skip option available?

**Screenshot:** Full question sequence; acknowledgment screen; first content screen (to assess visible impact)

**Framework check:** JTBD + Fogg MAP + Kano → Questions must probe the functional job. Visible impact of answers is the proof of personalization quality.

---

### 2. "For You" / Personalized Homepage Section
**Observe:**
- Present and prominent: above fold / below fold / absent?
- Relevance (1-5): do items match assumed user interests?
- Freshness: different items on second visit or same every time?
- Explanation label: "Based on your history" / "Trending in your interests" / none?
- Number of recommendations visible

**Screenshot:** Full homepage with "For You" section; zoomed individual rec with label; second-visit screenshot for freshness comparison

**Framework check:** HEART Engagement + Hook Variable Reward + Reforge Loop → Fresh "For You" recs drive return visits. Static recommendations break the product loop.

---

### 3. Recommendation Quality Throughout App
**Observe:**
- Recommendations after consuming content: "Related to this" / "Because you watched" / absent?
- Accuracy (1-5): do recs match content just viewed and user's broader history?
- Diversity (1-5): varied or same items repeated?
- Social signals included (ratings, views)?
- Click appetite: would a real user want to tap these?

**Screenshot:** Recommendations shown immediately after consuming content; two examples from different content to show variety (or lack of it)

**Framework check:** HEART Engagement + JTBD + Hook Variable Reward → Accurate, diverse recs increase session length. Irrelevant recs train users to ignore them.

---

### 4. Preference Settings & Customization
**Observe:**
- Settings location: easy to find (≤2 taps) / buried / absent?
- Available controls: language / content type / difficulty / interests / notification frequency?
- Impact: changing a preference visibly changes content (immediate or no feedback)?
- Reset option: can user clear personalization?
- Can user re-answer the onboarding personalization quiz?

**Screenshot:** Settings/preferences page; one preference changed with before/after content comparison if possible

**Framework check:** Nielsen #3 + Fogg MAP → User must be able to control their personalization. Easy control = high ability.

---

### 5. Watch/Read History & Content Tracking
**Observe:**
- History visible and accessible (≤2 taps)?
- "Continue watching/reading" present on homepage?
- History organization: chronological / by category / searchable?
- Can user delete history (individual items or all)?
- Privacy notice: is user told data is tracked?

**Screenshot:** History page; "Continue watching" section on homepage; delete option

**Framework check:** Reforge Loop + Hook Investment → History is the fuel for personalization. Resume feature is a direct engagement driver.

---

### 6. Difficulty/Content Scaffolding
**Observe:**
- Difficulty levels labeled (Beginner/Intermediate/Advanced) or all same level?
- Dynamic progression: does app suggest next level when current is completed?
- Learning path visible: can user see what's next?
- Remediation: if user struggles, does app offer easier content or help?

**Screenshot:** Difficulty labels on content; learning path or progression visual; next-level suggestion after completion

**Framework check:** Fogg MAP + JTBD + Hook → Content matched to skill level is more motivating and rewarding.

---

### 7. Cold Start (New User Experience)
**Observe:**
- What does a brand-new user with no history see on the homepage?
- Fallback strategy: trending / popular / category selection / blank?
- Time to first personalized recommendation: immediately (from quiz) / after 3-5 interactions / never?
- Seed personalization: upfront questions prime the engine?

**Screenshot:** Fresh-install first homepage (no history); fallback content quality

**Framework check:** JTBD + Fogg Motivation → Cold start is the hardest problem. Generic content for new users reduces motivation before the loop starts.

---

### 8. Personalization Transparency
**Observe:**
- Recommendation explanations: always / sometimes / never?
- Data collection notice: explicit / vague / absent?
- Opt-out option: can user disable personalization?
- Privacy/GDPR disclosure: clear?
- Dark patterns: is data collection opaque or clearly disclosed?

**Screenshot:** Recommendation with explanation label (or absence); privacy/data settings page

**Framework check:** Nielsen #1 + JTBD (Trust) + Kano (Must-Be in regulated markets) → Transparency builds trust. In India: RBI/TRAI sensitivity means users notice unexplained data use.

---

## SCORING RUBRIC

| Score | Label | Key Criteria |
|-------|-------|-------------|
| 9-10 | Best-in-class | JTBD onboarding + impact visible immediately + fresh "For You" above fold + recs accurate throughout + easy preference controls + resume feature + transparent explanations |
| 7-8 | Good | Onboarding mostly JTBD + "For You" visible + recs mostly relevant + history present + some controls + some transparency |
| 5-6 | Adequate | Minimal questions + "For You" exists but weak/generic + recs hit-or-miss + history present but limited + controls buried |
| 3-4 | Poor | No onboarding personalization + no "For You" + recs absent or generic trending + no resume + no controls |
| 1-2 | Critical | Data collected but ignored + dark patterns + no disclosure + all users identical + cold start broken |

**Scoring method:** Start at 5. Add: +1 JTBD questions with visible impact, +1 "For You" above fold and relevant, +1 recs accurate and fresh throughout, +1 preference controls easy to find, +1 resume/"continue" feature works, +1 cold start handled with seed personalization, +1 explanation labels on recs. Deduct: -2 no personalization at all, -1 quiz asked but ignored, -1 "For You" absent or always same, -1 no resume, -1 dark pattern or no privacy notice. Cap at 1-10.

## KEY ISSUE PATTERNS

1. **No Personalization Upfront — Generic for All** → Detect: No onboarding questions; all users see identical homepage regardless of interests. Framework: JTBD, Kano, Fogg MAP. Fix: Add 5-7 JTBD questions (goal, experience, time commitment); change first content screen based on answers; acknowledge selection visibly.

2. **"For You" Absent or Always the Same** → Detect: No dedicated recommendations section, or section shows same items on every visit. Framework: HEART Engagement, Hook Variable Reward, Reforge Loop. Fix: Build "For You" above fold; refresh daily with collaborative filtering; add explanation label.

3. **Irrelevant Recommendations** → Detect: Post-content recs have no relationship to content just viewed or user history; CTR estimate <5%. Framework: JTBD, HEART Task Success, Hook. Fix: Use content-based + collaborative filtering; add "helpful?" feedback; track CTR target >15%.

4. **No Resume / Continue Feature** → Detect: User returns to app after partial consumption; must manually find where they left off. Framework: Reforge Loop, Hook Investment. Fix: "Continue watching/reading" on homepage; resume from last position automatically.

5. **Cold Start Broken** → Detect: Fresh install shows blank recommendations or generic trending unrelated to user's stated interests. Framework: JTBD, Fogg Motivation. Fix: Seed from onboarding questions; if skipped, show popular content by category until behavioral data accumulates.
