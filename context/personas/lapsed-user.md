# Persona: Lapsed User (30+ Days Inactive)

> User downloaded 30+ days ago, didn't engage, returning now.

## What to Test Across All Modules

| Module | Persona-Specific Focus |
|--------|----------------------|
| **Onboarding** | Is re-onboarding offered? Or same as first visit? "What's new"? |
| **Homepage & IA** | Does app leverage prior data? "Welcome back" messaging? |
| **Discovery & Search** | Are prior interests remembered? Or reset to generic? |
| **Personalization** | Is prior personalization retained? Updated content since last visit? |
| **Content Consumption** | Prior progress saved? Can user resume? |
| **Engagement & Retention** | Win-back hooks? Streak recovery? Re-engagement incentives? |
| **Monetization** | Win-back pricing? Special offers for returners? |
| **AI Integration** | Does AI acknowledge return? Adapted recommendations? |
| **Cognitive Load** | Is the return experience simpler (user already knows the app)? |
| **Persona-Intent** | Does app re-probe intent? User's job may have changed |

## Key Metrics

- **Primary**: Lapsed user reactivation rate
- **Recognition** (does app know user is returning, not new?)
- **Win-back offers** present and quality
- **Prior data retention** (preferences, progress, content)

## Scoring Focus

Score higher when: app recognizes returning user, offers win-back, retains prior data, shows "what's new."
Score lower when: treats lapsed user as brand new, no re-engagement strategy, prior progress lost.
